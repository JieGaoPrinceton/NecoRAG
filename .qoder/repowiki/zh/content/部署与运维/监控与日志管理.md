### Docker编排配置详解
- **Grafana服务配置**
  - 镜像：grafana/grafana:latest

### Grafana数据源配置
- **数据源类型**

## 性能考虑
- 指标采集粒度
  - 按模块维度（感知/记忆/检索/巩固/响应）拆分指标，便于定位瓶颈
  - 在关键路径（向量检索、重排序、生成、可视化）埋点耗时
- 资源监控
  - CPU/内存：结合系统监控（如 cAdvisor/Prometheus Node Exporter）与进程内采样
  - 磁盘：监控向量库、图数据库、日志目录的使用率
  - 网络：监控 LLM 推理与外部服务调用的 RTT 与错误率
- **新增** Grafana监控优化
  - 指标数据保留策略：短期高频数据（15天），长期低频数据（90天）
  - 图表刷新频率：生产环境建议30秒-1分钟刷新
  - 数据聚合：对高频指标进行聚合处理，减少存储压力
- 性能基线
  - 建立端到端延迟、检索命中率、生成质量等基线，定期回归对比
- 容量规划
  - 基于检索 top-k、向量维度、实体数量估算 L2/L3 存储与索引开销
  - 依据会话并发与响应长度估算 L1 缓存与 LLM 推理成本

## 故障排查指南
- Dashboard 无法访问
  - 检查端口占用与防火墙；参考 DASHBOARD_GUIDE 中的常见问题章节
- 统计信息不更新
  - 确认前端定时轮询 /api/stats 是否正常；检查后端日志级别
- 配置变更未生效
  - 确认已激活目标 Profile，并在业务侧重新初始化对应模块
- 模块参数校验失败
  - 确认模块名在允许列表内（whiskers/memory/retrieval/grooming/purr）
- **新增** Grafana监控问题排查
  - Grafana无法访问：检查端口映射和容器状态
  - 数据源连接失败：确认Prometheus服务可用性和URL配置
  - 图表无数据显示：检查指标采集器是否正常运行
  - 容器启动失败：查看Docker日志和环境变量配置

### 告警规则与异常检测建议
- 触发条件示例
  - 查询/会话数在 5 分钟内下降 50%
  - 检索平均延迟超过阈值且持续 3 个周期
  - 内存使用率超过阈值且增长趋势明显
  - **新增** Grafana告警规则
    - CPU使用率连续5分钟超过80%
    - 内存使用率超过70%
    - 磁盘空间剩余小于10%
    - 应用响应时间超过2秒
- 自动恢复
  - 重启异常实例、切换备用节点、降级非关键功能

### 日志查询与分析工具使用指南
- 日志目录
  - NecoRAGConfig 中提供 log_dir，默认位于 ./logs
- 建议
  - 使用结构化日志（JSON），统一时间戳与级别
  - 通过 OpenSearch/ELK 或 OTel Collector 进行采集与检索
  - 建立常用查询模板（按模块、按会话、按错误类型）
- **新增** Grafana日志分析
  - 结合Loki或Elasticsearch进行日志聚合
  - 创建日志仪表板，实时监控错误率和异常模式
  - 设置日志告警规则，及时发现系统问题
