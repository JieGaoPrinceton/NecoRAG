# NecoRAG 项目统计工具使用说明

## 📊 工具概述

`project_statistics.py` 是一个用于统计 NecoRAG 项目文件数量和代码行数的 Python 工具。它可以提供详细的项目结构分析和代码质量指标。

## 🚀 快速使用

### 基本用法
```bash
# 统计当前目录项目
python tools/project_statistics.py

# 统计指定目录
python tools/project_statistics.py -p /path/to/project

# 导出CSV报告
python tools/project_statistics.py -o statistics.csv

# 组合使用
python tools/project_statistics.py -p src -o src_stats.csv
```

## 📋 命令行参数

| 参数 | 简写 | 说明 | 默认值 |
|------|------|------|--------|
| `--path` | `-p` | 项目根目录路径 | 当前目录 |
| `--output` | `-o` | 导出CSV文件路径 | 无 |
| `--verbose` | `-v` | 显示详细信息 | False |

## 📈 输出信息详解

### 基本统计
- **总文件数**: 项目中所有文件的总数
- **代码文件**: 可读取的源代码文件数量
- **二进制文件**: 图片、压缩包等二进制文件数量
- **其他文件**: 配置文件、文档等其他类型文件
- **目录数量**: 项目中的目录层级数量

### 代码行数统计
- **总行数**: 所有代码文件的总行数
- **代码行数**: 实际的代码逻辑行数
- **空行数**: 空白行数量
- **注释行数**: 注释行数量
- **行数比例**: 各类行数占总行数的百分比

### 文件类型分布
按文件扩展名统计各类文件的数量和占比，显示前15个最常见的文件类型。

### 代码行数分布
按文件类型统计代码行数，计算平均每文件行数，显示前10个主要类型。

## 📁 支持的文件类型

### 代码文件 (`.py`, `.js`, `.ts`, `.java`, `.cpp` 等)
- 统计代码行、注释行、空行
- 支持多种编程语言的注释语法识别

### 文档文件 (`.md`, `.rst`, `.txt`)
- 统计总行数
- 归类为其他文件类型

### 配置文件 (`.yaml`, `.json`, `.xml`)
- 统计总行数
- 部分支持注释行识别

### 二进制文件 (`.png`, `.jpg`, `.pdf` 等)
- 仅统计文件数量
- 不计入行数统计

## 📝 注意事项

1. 工具会自动忽略常见的临时文件和构建产物
2. 二进制文件只统计数量，不计入行数
3. 注释识别针对不同语言有专门处理
4. 大文件可能影响统计速度
5. 建议在项目根目录运行以获得完整统计
