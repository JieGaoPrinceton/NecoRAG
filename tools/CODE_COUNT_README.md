# NecoRAG 代码统计工具使用说明

## 📊 工具概述

`code-count.py` 是 NecoRAG 项目的专用代码行数统计工具，可以：
- 统计项目总文件数和各类文件数量
- 统计代码行数、空行数、注释行数
- 自动读取项目版本号（从 VERSION 文件）
- 添加时间戳记录
- 导出 Markdown 格式报告

## 🚀 快速使用

### 基本用法

```bash
# 在项目根目录运行
python tools/code-count.py

# 或在 tools 目录运行
cd tools
python code-count.py -p ..
```

### 命令行参数

| 参数 | 简写 | 说明 | 默认值 |
|------|------|------|--------|
| `--path` | `-p` | 项目根目录路径 | `..` (项目根目录) |
| `--output` | `-o` | 导出 Markdown 报告 | 不导出 |
| `--verbose` | `-v` | 显示详细信息 | False |

## 📊 输出内容详解

### 终端输出包含：

1. **版本信息**
   - 自动从 `VERSION` 文件读取版本号
   - 格式：`v{version}`

2. **时间戳**
   - 精确到秒的统计时间
   - 格式：`YYYY-MM-DD HH:MM:SS`

3. **文件统计**
   - 总文件数
   - 代码文件数（.py, .js, .ts 等）
   - 二进制文件数（.png, .jpg, .pdf 等）
   - 其他文件数

4. **代码行数统计**
   - 总行数
   - 代码行数（实际逻辑代码）
   - 空行数
   - 注释行数
   - 各类型行数占比

5. **文件类型分布**
   - Top 15 文件类型
   - 每种类型的文件数量和占比
   - 平均每个文件的行数

6. **代码行数分布**
   - Top 10 代码类型
   - 每种类型的总行数和占比
   - 平均每个文件的行数

### Markdown 报告包含：

- 完整的统计表格
- 美观的 Markdown 格式
- 版本号和时问戳
- 可用于文档归档

## ⚠️ 注意事项

1. **版本号来源**
   - 确保项目根目录有 `VERSION` 文件
   - 如果找不到 VERSION 文件，显示为 "unknown"

2. **统计准确性**
   - 多行注释可能只统计第一行
   - 字符串中的注释标记可能被误计
   - 建议作为参考指标而非绝对值

3. **性能考虑**
   - 大文件（>10MB）可能影响统计速度
   - 大量小文件也会增加处理时间
   - 建议使用 `-v` 参数查看详细进度
