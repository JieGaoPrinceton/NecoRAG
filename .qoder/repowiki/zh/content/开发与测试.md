## 核心组件
- 配置管理：统一的 NecoRAGConfig 与各层子配置，支持文件与环境变量加载，提供开发/生产/最小化预设。
- 记忆管理：MemoryManager 统一管理 L1/L2/L3 三层记忆，支持向量检索、实体图谱与记忆衰减。
- 自适应检索：AdaptiveRetriever 集成 HyDE、重排序、领域权重与早停机制。
- 精炼代理：RefinementAgent 实现 Generator-Critic-Refiner 闭环与幻觉检测。
- Dashboard：ConfigManager 提供 Profile 的创建、切换、导入导出与参数管理。

### 自适应检索器（HyDE/重排序/早停/领域权重）
- 多路检索：向量检索 + 图谱检索（实体）。
- 结果融合：使用倒数秩融合策略。
- 重排序：基于 BGE-Reranker-v2。
- 领域权重：可选的复合权重计算与查询增强。
- 早停机制：基于置信度阈值与边际收益的智能终止。

```mermaid
flowchart TD
Start(["开始检索"]) --> QAnalyze["分析查询<br/>识别类型/复杂度/实体"]
QAnalyze --> MultiSearch["多路检索<br/>向量 + 图谱"]
MultiSearch --> Fusion["结果融合<br/>RRF"]
Fusion --> Rerank["重排序<br/>BGE-Reranker-v2"]
Rerank --> DomainWeight{"启用领域权重?"}
DomainWeight --> |是| ApplyDW["应用领域权重<br/>关键字/时间/领域"]
DomainWeight --> |否| FilterMin["过滤最低分数"]
ApplyDW --> FilterMin
FilterMin --> EarlyTerm["早停判断<br/>置信度阈值/边际收益"]
EarlyTerm --> |满足| ReturnTopK["返回 Top-K 结果"]
EarlyTerm --> |不满足| Expand["扩大搜索范围继续"]
Expand --> Rerank
```

### 精炼代理（Generator-Critic-Refiner 闭环）
- 迭代验证：生成 → 批判 → 修正 → 幻觉检测。
- 置信度控制：未通过时降低置信度，达到阈值才返回。
- 异步任务：知识固化与记忆修剪（需 MemoryManager）。

```mermaid
sequenceDiagram
participant U as "调用方"
participant RA as "RefinementAgent"
participant GEN as "Generator"
participant CRIT as "Critic"
participant REF as "Refiner"
participant DET as "HallucinationDetector"
U->>RA : process(query, evidence)
RA->>GEN : generate(query, evidence)
GEN-->>RA : 初始答案(含置信度/引用)
loop 最多 max_iterations 次
RA->>CRIT : critique(answer, evidence)
CRIT-->>RA : 批判结果
RA->>DET : detect(answer.content, evidence)
DET-->>RA : 幻觉报告
alt 通过验证
RA-->>U : 返回 RefinementResult
else 未通过
RA->>REF : refine(answer, critique)
REF-->>RA : 修正后的答案
RA->>RA : 若幻觉则降低置信度
end
end
RA-->>U : 返回最终结果(可能为空/兜底)
```

## 性能考量
- 检索性能：通过早停机制在达到置信度阈值时立即返回，减少不必要的重排序与领域权重计算；合理设置 top_k 与 rerank_top_k。
- 记忆衰减：通过权重衰减与归档策略降低上下文规模，提升检索效率。
- 重排序与领域权重：仅在必要时启用，避免对简单查询造成额外开销。
- 容器化与资源：Docker Compose 提供 Redis/Qdrant/Neo4j/Ollama/Grafana 等服务编排，建议按需启用与资源限制。

## 故障排除指南
- 环境与依赖
  - Python 版本不符：确认使用 3.9+。
  - 依赖缺失：执行安装命令，参考 requirements.txt 与 pyproject.toml 的 dev 依赖。
- Dashboard 启动失败
  - 端口占用：检查 8000 端口，必要时更换端口。
  - 服务健康检查：容器编排中包含健康检查，确认依赖服务（Redis/Qdrant/Neo4j/Ollama）可用。
- 模块导入与基础功能
  - 使用导入测试脚本验证模块导入与基础初始化。
  - 运行完整示例，观察各层协同行为。
- 配置问题
  - 通过 ConfigManager 管理 Profile，确保活动 Profile 正确设置。
  - 使用环境变量覆盖关键配置（如 LLM/向量/图数据库提供商与地址）。
