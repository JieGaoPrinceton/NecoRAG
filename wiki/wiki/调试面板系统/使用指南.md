# NecoRAG 调试面板使用指南

## 🎯 概述

NecoRAG 调试面板是一个功能强大的可视化调试工具，专为展示AI系统的"思维路径"而设计。它提供了实时监控、路径分析、性能优化等全方位的调试能力。

## 🚀 快速开始

### 启动调试面板

```bash
# 方法1: 使用启动脚本
cd /Users/ll/NecoRAG
python tools/start_dashboard.py

# 方法2: 直接运行
cd src/dashboard
python server.py
```

访问地址: http://localhost:8000

### 基本使用流程

1. **创建调试会话**
```python
from src.dashboard.debug import DebugSession

# 创建新的调试会话
session = DebugSession(
    query="用户查询内容",
    user_id="user_123"
)
```

2. **添加检索步骤**
```python
from src.dashboard.debug import RetrievalStep

step = RetrievalStep(
    name="向量检索",
    description="执行向量相似度搜索",
    input_data={"query": "用户问题"},
    metrics={"latency": 0.5, "recall": 0.85}
)

session.add_retrieval_step(step)
```

3. **添加证据信息**
```python
from src.dashboard.debug import EvidenceInfo

evidence = EvidenceInfo(
    source="vector",
    content="相关文档片段",
    relevance_score=0.92,
    metadata={"chunk_id": "doc_123"}
)

session.add_evidence(evidence)
```

4. **完成会话**
```python
session.complete_session({
    "accuracy": 0.95,
    "confidence": 0.88
})
```

## 📊 核心功能模块

### 1. 思维路径可视化
- **时间轴展示**: 清晰展示检索过程的时间顺序
- **证据来源**: 可视化不同来源的证据信息
- **推理过程**: 图表化展示AI的思考逻辑

### 2. 实时监控
- **WebSocket推送**: 实时更新调试信息
- **性能指标**: CPU、内存、响应时间监控
- **连接状态**: WebSocket连接健康检查

### 3. 路径分析工具

```python
from src.dashboard.debug import PathAnalyzer

analyzer = PathAnalyzer()

```

```python
tuner = ParameterTuningPanel()
```

### 5. A/B测试框架
```python
from src.dashboard.debug import ABTestingFramework

framework = ABTestingFramework()

# 创建测试
test_config = framework.create_test(
    name="检索算法对比",
    variants=[
        {"name": "variant_a", "weight": 0.5, "params": {"algorithm": "bm25"}},
        {"name": "variant_b", "weight": 0.5, "params": {"algorithm": "hybrid"}}
    ],
    metrics=["accuracy", "latency", "user_satisfaction"]
)

# 运行测试
results = framework.run_test(test_config, test_data)
report = framework.generate_report(results)
```

## 🔧 API接口

### 调试会话API
```
POST /api/debug/sessions          # 创建调试会话
GET /api/debug/sessions/{id}      # 获取会话详情
POST /api/debug/sessions/{id}/complete  # 完成会话
```

### WebSocket端点
```
/ws/thinking-path/{session_id}    # 思维路径实时推送
/ws/app                          # 应用状态推送
```

### 性能监控API
```
GET /api/debug/stats              # 获取调试统计
GET /api/debug/stats/dashboard    # 获取仪表板数据
GET /api/debug/health             # 健康检查
```

```python
error_handler.add_notification_callback(error_callback)

# 装饰器使用
@handle_errors({"context": "my_function"})
```

## 🆘 故障排除

### 常见问题

**Q: WebSocket连接失败**
A: 检查网络连接和防火墙设置，确认服务器正常运行

**Q: 性能监控数据显示异常**
A: 确认psutil依赖已安装，检查系统权限

**Q: 调试会话创建失败**
A: 检查必填参数，确认系统资源充足
